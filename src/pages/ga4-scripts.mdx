## Scripts

We use 3rd party scripts to gather data and provide insights into user behavior, website performance, and audience engagement for the purpose of optimizing the website's effectiveness and making data-driven decisions.

### Facebook Pixel

Facebook Pixel scripts are snippets of code provided by Facebook that are placed on a website to collect data and track user interactions. When a user visits a web page that has the Facebook Pixel script installed, the script sends information back to Facebook, allowing businesses to measure the effectiveness of their Facebook advertising campaigns and optimize their targeting strategies.

```html
<!-- Facebook Pixel Code -->
<script type="text/javascript">
  !(function (f, b, e, v, n, t, s) {
    if (f.fbq) return
    n = f.fbq = function () {
      n.callMethod ? n.callMethod.apply(n, arguments) : n.queue.push(arguments)
    }
    if (!f._fbq) f._fbq = n
    n.push = n
    n.loaded = !0
    n.version = '2.0'
    n.queue = []
    t = b.createElement(e)
    t.async = !0
    t.src = v
    s = b.getElementsByTagName(e)[0]
    s.parentNode.insertBefore(t, s)
  })(
    window,
    document,
    'script',
    'https://connect.facebook.net/en_US/fbevents.js'
  )
  fbq('init', '1675948675780825') // Insert your pixel ID here.
  fbq('track', 'PageView')
</script>
<noscript
  ><img
    height="1"

    style="display:none"
    src="https://www.facebook.com/tr?id=1675948675780825&ev=PageView&noscript=1"
/></noscript>

```

This code enables tracking and integration capabilities provided by the Outfunnel platform for marketing and sales automation purposes.

```html
<!-- outfunnel start -->
<script type="text/javascript">
  window.OFID = '61c5ba3eeb73af43f3a3446c'
```

### Mailchimp Goals

This code dynamically loads and executes the Mailchimp goal.min.js script, which is responsible for tracking and goal tracking functionality. This script facilitates the tracking of specific user actions or events on the webpage and sends relevant data to Mailchimp for analysis or marketing automation purposes.

```html
<!--mail chimp start -->
<script defer type="text/javascript">
  var $mcGoal = { settings: { uuid: '9cd2a1ffa021754103be84a32', dc: 'us1' } }
  ;(function () {
    var sp = document.createElement('script')
    sp.type = 'text/javascript'
    sp.async = true
    sp.defer = true
    sp.src =
      ('https:' == document.location.protocol
        ? 'https://s3.amazonaws.com/downloads.mailchimp.com'
        : 'http://downloads.mailchimp.com') + '/js/goal.min.js'
    var s = document.getElementsByTagName('script')[0]
    s.parentNode.insertBefore(sp, s)
  })()
</script>
<!--mail chimp end -->
```

### Gravity Form - Listener - Newsletter

This code helps capture form submission data and trigger a custom event when specific forms (identified by their formIDs) are successfully submitted using Gravity Forms.

```html
<!-- gravity forms start -->
<script type="text/javascript">
  jQuery(document).ready(function () {
    jQuery(document).bind(
      'gform_confirmation_loaded',
      function (event, formID) {
        if (formID == 6 || formID == 14) {
          window.dataLayer = window.dataLayer || []
          window.dataLayer.push({
            event: 'formSubmissionNewsletter',
            formID: formID,
          })
        }
      }
    )
  })
</script>
<!-- gravity forms end -->
```

### Gravity Form - Listener

This code helps capture form submission data and triggers a custom event when a specific form (identified by its formID) with the ID of 10 is successfully submitted using Gravity Forms. The purpose is to track and send information about the form submission to the window.dataLayer array, enabling further processing or tracking.

```html
<!-- gravity forms start -->
<script type="text/javascript">
  jQuery(document).ready(function () {
    jQuery(document).bind(
      'gform_confirmation_loaded',
      function (event, formID) {
        if (formID == 10) {
          window.dataLayer = window.dataLayer || []
          window.dataLayer.push({
            event: 'formSubmission',
            formID: formID,
          })
        }
      }
    )
  })
</script>
<!-- gravity forms end -->
```

## Triggers

Triggers in Google Tag Manager are responsible for determining when tags should be fired or executed on a website or mobile app. They define the conditions that need to be met for a particular tag to be triggered and initiate its associated tracking or functionality. Triggers can be based on various factors, such as specific page URLs, clicks on specific elements, form submissions, time delays, or even custom events. By configuring triggers in Google Tag Manager, you have fine-grained control over when and where your tags are activated, ensuring that tracking and other scripts are implemented precisely at the desired moments during user interactions.

## Variables

Variables in Google Tag Manager serve as placeholders or dynamic values that are used to capture and retrieve data from a website or mobile app. They provide a way to access and manipulate information such as page URLs, click events, form inputs, or custom data elements. Variables act as inputs for tags and triggers, allowing you to pass dynamic values or extract specific data points that are necessary for tracking, customization, or other functionality.
